## ✨ 新增功能

### 1. 🔧 动态配置系统
**问题：** 之前服务器IP地址硬编码在环境变量中，不同网络环境需要重新构建

**解决方案：**
- ✅ 服务器IP地址现在从配置文件动态加载
- ✅ 添加了配置管理界面，可在浏览器中直接修改
- ✅ 配置文件位置：`frontend/public/config.json`

**使用方法：**
1. 点击页面右上角 **「⚙️ 设置」** 按钮
2. 在弹出的配置窗口中修改服务器API地址
3. 点击 **「保存」** 按钮
4. 页面将自动刷新，新配置立即生效

**快速配置：**
- **本地开发：** `http://localhost:8000/api/v1`
- **局域网访问：** `http://192.168.1.100:8000/api/v1`
- **自定义IP：** 输入你的服务器IP地址

## 🔧 技术细节

### 新增文件
1. **`backend/app/api/config.py`** - 配置API路由
   - `GET /api/v1/config` - 获取配置
   - `POST /api/v1/config` - 更新配置
   - `POST /api/v1/config/reset` - 重置为默认配置

2. **`frontend/src/lib/config.ts`** - 配置加载器
   - 从 `/config.json` 动态加载配置
   - 配置缓存机制
   - 失败时自动使用默认配置

3. **`frontend/src/components/ConfigManager.tsx`** - 配置管理组件
   - 可视化配置界面
   - URL格式验证
   - 快速配置按钮

4. **`frontend/public/config.json`** - 配置文件
   ```json
   {
     "apiUrl": "http://192.168.1.100:8000/api/v1",
     "version": "1.0.0",
     "lastUpdate": "2026-03-12"
   }
   ```

### 修改文件
- **`backend/app/main.py`** - 注册配置API路由
- **`frontend/src/lib/api.ts`** - 支持动态API地址（拦截器）

- **`frontend/src/components/BenchCard.tsx`** - UI样式优化

## 🚀 部署指南

### 方式1：使用默认配置
直接启动服务，系统将使用 `public/config.json` 中的配置

### 方式2：自定义配置
1. 编辑 `frontend/public/config.json`：
   ```json
   {
     "apiUrl": "http://你的服务器IP:8000/api/v1",
     "version": "1.0.0",
     "lastUpdate": "2026-03-12"
   }
   ```

2. 重新构建前端：
   ```bash
   cd frontend
   npm run build
   ```

### 方式3：运行时修改
1. 启动服务后，在浏览器中点击「⚙️ 设置」
2. 修改API地址并保存
3. 页面自动刷新，配置立即生效
