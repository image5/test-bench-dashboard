# 测试用例匹配指南

本文档详细说明测试用例匹配器的使用方法和最佳实践。

## 快速开始

### 基本用法

```
/testcase-matcher-skill 客户用例.xlsx 内部用例.xlsx
```

Agent 会自动：
1. 解析两个 Excel 文件
2. 智能识别客户用例的结构（子表、列名）
3. 执行语义匹配
4. 生成匹配报告

### 输出格式选择

```
/testcase-matcher-skill 客户用例.xlsx 内部用例.xlsx --output excel
/testcase-matcher-skill 客户用例.xlsx 内部用例.xlsx --output markdown
```

## 文件格式要求

### 客户用例文件

客户用例文件格式灵活，系统会自动识别：

**支持的子表结构：**
- 系统会自动识别哪个子表包含测试用例
- 跳过目录页、汇总表等非用例内容

**支持的列名（自动识别）：**

| 用途 | 可能的列名 |
|-----|-----------|
| 用例名称 | 用例名称、测试用例、Case Name、测试项、功能点、测试点、测试内容、测试场景、用例标题、测试名称、用例编号 |
| 测试步骤 | 测试步骤、操作步骤、步骤、Step、测试过程、操作过程、执行步骤、测试流程、操作描述、测试操作、执行过程、用例步骤 |
| 预期结果 | 预期结果、期望结果、预期、Expected、期望、预期输出、预期表现、期望输出 |
| 预置条件 | 预置条件、前置条件、前提条件、Precondition、前置、准备条件、测试前提 |

**识别优先级：**
1. 优先选择包含"步骤"关键词的列作为步骤列
2. 选择能唯一标识用例的列作为名称列
3. 如果存在多个候选，选择数据最完整、格式最规范的列

### 内部用例文件

内部用例文件需要包含以下固定字段：

| 字段名 | 说明 | 必填 |
|-------|------|-----|
| 用例_名称 | 用例标识 | 是 |
| 用例_预置条件 | 执行前准备 | 否 |
| 用例_测试步骤 | 核心操作步骤 | 是 |
| 用例_预期结果 | 期望输出 | 否 |

## 匹配算法说明

### 匹配策略

系统采用**语义级别匹配**，而非简单的关键词匹配：

1. **步骤语义分析**
   - 提取操作动作、操作对象、预期结果
   - 计算语义相似度
   - 关注核心测试意图

2. **多对多映射**
   - 一个客户用例可能对应多个内部用例
   - 一个内部用例可能对应多个客户用例
   - 每个映射都有独立的置信度

### 置信度判定

| 置信度 | 相似度阈值 | 说明 | 建议 |
|-------|-----------|------|------|
| 高 | ≥ 60% | 步骤语义高度相似，操作对象一致 | 可直接采用 |
| 中 | 40%-60% | 核心操作一致，细节有差异 | 建议快速确认 |
| 低 | 20%-40% | 仅主题相关，步骤差异较大 | 需人工确认 |

## 输出报告解读

### Excel 报告结构

生成的 Excel 文件包含 4 个工作表：

1. **匹配汇总**
   - 所有匹配关系的详细列表
   - 包含置信度和匹配理由

2. **客户用例视角**
   - 以客户用例为主视角
   - 显示每个客户用例对应的所有内部用例
   - 便于确认客户需求是否被覆盖

3. **内部用例视角**
   - 以内部用例为主视角
   - 显示每个内部用例对应的所有客户用例
   - 便于评估内部用例的价值

4. **统计信息**
   - 匹配率、覆盖率等关键指标
   - 高/中/低置信度分布

### 关键指标说明

| 指标 | 计算方式 | 意义 |
|-----|---------|------|
| 客户用例匹配率 | 已匹配客户用例 / 客户用例总数 | 客户需求覆盖程度 |
| 内部用例覆盖率 | 已匹配内部用例 / 内部用例总数 | 内部用例利用率 |
| 高置信度占比 | 高置信度匹配数 / 总匹配数 | 匹配质量 |

## 最佳实践

### 提高匹配准确率

1. **规范用例编写**
   - 测试步骤描述清晰、完整
   - 使用标准术语
   - 避免模糊表述

2. **统一用例粒度**
   - 客户用例和内部用例粒度尽量一致
   - 过粗或过细都会影响匹配效果

3. **定期维护内部用例库**
   - 清理过时用例
   - 补充新功能用例
   - 保持用例库的完整性

### 处理特殊情况

1. **新需求识别**
   - 未匹配的客户用例可能是新需求
   - 建议创建新的内部用例

2. **冗余用例处理**
   - 未被覆盖的内部用例可能是冗余的
   - 建议评估是否删除或合并

3. **低置信度匹配**
   - 需要人工确认
   - 确认结果可用于优化匹配算法

## 常见问题

### Q: 为什么某些明显相似的用例没有匹配？

A: 可能原因：
- 步骤描述差异过大
- 使用了不同的术语
- 用例粒度不一致

建议：检查用例描述，或调整匹配阈值。

### Q: 如何处理大量用例？

A: 建议：
- 分批处理（每批不超过 500 条）
- 先处理核心功能用例
- 使用 Excel 报告便于筛选

### Q: 匹配结果如何复用？

A: 确认后的匹配结果可以：
- 作为历史记录保存
- 用于后续匹配的参考
- 作为训练数据优化算法

## 进阶用法

### 结合大模型增强

当前版本的匹配基于关键词相似度，可以结合大模型进行语义增强：

```
/testcase-matcher-skill 客户用例.xlsx 内部用例.xlsx --enhance-llm
```

此模式会调用大模型进行深度语义分析，提高匹配准确率，但会增加处理时间。

### 自定义匹配规则

可以在 `assets/custom_rules.json` 中定义自定义匹配规则：

```json
{
  "keyword_weights": {
    "登录": 1.5,
    "支付": 1.5,
    "订单": 1.3
  },
  "exclude_patterns": [
    "测试环境",
    "回归测试"
  ]
}
```
