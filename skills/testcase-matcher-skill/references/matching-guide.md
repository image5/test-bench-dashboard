# 测试用例匹配指南

本文档说明如何使用测试用例匹配器。

## 快速开始

### 1. 激活 Skill

在对话中输入：

```
/testcase-matcher-skill
```

### 2. 提供参数

Agent 会询问以下信息：

| 参数 | 说明 | 示例 |
|-----|------|------|
| 客户文件路径 | 客户测试用例 Excel 文件 | `D:\客户用例.xlsx` |
| 客户子表名称 | 包含用例的子表 | `测试用例` |
| 客户用例名称列 | 用例标识所在的列 | `B` 或 `用例名称` |
| 客户测试步骤列 | 测试步骤所在的列 | `D` 或 `测试步骤` |
| 内部文件路径 | 内部测试用例 Excel 文件 | `D:\内部用例.xlsx` |
| 内部子表名称 | 包含用例的子表 | `Sheet1` |
| 内部用例名称列 | 用例标识所在的列 | `A` 或 `用例_名称` |
| 内部测试步骤列 | 测试步骤所在的列 | `C` 或 `用例_测试步骤` |

### 3. 获取报告

Agent 会生成匹配报告 Excel 文件。

## 列规格说明

支持三种方式指定列：

### 1. 列字母（推荐）

使用 Excel 列字母：`A`, `B`, `C`, `D` ...

```
客户用例名称列：B
客户测试步骤列：D
```

### 2. 列名

使用列标题中的关键词：

```
客户用例名称列：用例名称
客户测试步骤列：测试步骤
```

### 3. 列序号

使用数字（从 1 开始）：

```
客户用例名称列：2
客户测试步骤列：4
```

## 输出报告结构

生成的 Excel 文件包含 5 个工作表：

### 1. 匹配结果（客户视角）

| 序号 | 客户用例名称 | 匹配的内部用例 | 匹配程度 |
|-----|------------|--------------|---------|
| 1 | 登录功能测试 | TC_LOGIN_001(高); TC_LOGIN_002(高) | 高 |
| 2 | 支付流程测试 | TC_PAY_001(中) | 中 |
| 3 | 新功能测试 | 无匹配 | 无匹配 |

### 2. 内部用例覆盖（内部视角）

| 序号 | 内部用例名称 | 对应的客户用例 | 覆盖程度 |
|-----|------------|--------------|---------|
| 1 | TC_LOGIN_001 | 登录功能测试(高) | 高 |
| 2 | TC_OLD_001 | 未被覆盖 | 无 |

### 3. 未匹配项

列出未找到匹配的客户用例和未被覆盖的内部用例。

### 4. 统计信息

- 客户用例总数
- 内部用例总数
- 匹配率
- 覆盖率
- 高/中/低匹配数

### 5. 详细匹配记录

每条匹配关系的详细信息，包括相似度分数。

## 匹配程度说明

| 程度 | 相似度 | 说明 | 建议 |
|-----|-------|------|------|
| 高 | ≥50% | 步骤高度相似，核心操作一致 | 可直接采用 |
| 中 | 30%-50% | 步骤部分相似，有共同点 | 建议快速确认 |
| 低 | 15%-30% | 仅主题相关，差异较大 | 需人工确认 |

## 注意事项

1. **文件格式**：仅支持 `.xlsx` 和 `.xls` 格式
2. **数据质量**：确保用例名称和步骤列有数据，空行会自动跳过
3. **大数据量**：超过 500 条用例时，处理时间会增加
4. **人工复核**：低度匹配建议人工确认
5. **输出路径**：默认输出到当前目录，可指定输出路径

## 常见问题

### Q: 列名匹配不上怎么办？

A: 使用列字母（A, B, C）是最可靠的方式。

### Q: 子表名称不知道怎么办？

A: 可以留空，系统会使用第一个子表。或者打开 Excel 查看底部标签。

### Q: 匹配结果不准确怎么办？

A: 
- 检查是否选择了正确的步骤列
- 确保步骤描述清晰完整
- 低度匹配需要人工确认

### Q: 如何处理多对多映射？

A: 这是正常的。一个客户用例可能覆盖多个内部用例的功能点，反之亦然。报告会同时展示两个视角。
